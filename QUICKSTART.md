# Cowork System — Quickstart Guide

This guide explains how to replicate the five-file Cowork System scaffold in a new project.

---

## What the system is

Five files that give Claude persistent context across sessions — like an onboarding doc for a new hire that you never have to repeat:

| File | Role |
|------|------|
| `CLAUDE.md` | Project identity + Claude's behavioral config |
| `Instructions.md` | SOPs/workflows tailored to how you work |
| `memory.md` | Running knowledge log — what's been learned, changed, decided |
| `PROTOCOL.md` | When and how to update memory |
| `QUICKSTART.md` | This file — how to replicate the system |

---

## How to set it up in a new project

1. Open Cowork and select or create the target project folder
2. Type `/project-deploy` in the chat
3. Answer the interview questions (2 rounds, ~5 minutes)
4. Claude generates all five files, customized to your project

---

## How to use the system day-to-day

**Starting a session:**
> "Read my context files and tell me what project we're working on and what my workflows are."

**Running a workflow:**
> "Let's do a Copy & Content Writing session for the homepage hero section."

**Updating memory after a session:**
> "Update memory.md — we decided to use Tailwind for all styling."

---

## Verification test

After setup, start a new session and say:
> "Read my context files and tell me what project we're working on and what my workflows are."

Claude should respond with the project name, purpose, Claude's role, and the active workflow list — without you repeating any of it.
